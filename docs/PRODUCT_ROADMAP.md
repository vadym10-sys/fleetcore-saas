# FleetCore SaaS Product Roadmap

## Что уже критично для продаж B2B

- Полноценная регистрация компании через email владельца.
- Изоляция данных по tenant/company.
- Авторизация через access token.
- Автопарк, клиенты, аренды, финансы, GPS, ТО и документы.
- Загрузка файлов в систему и привязка к авто, клиентам, договорам и депозитам.

## Ближайшие улучшения продукта

1. Онбординг после регистрации
   - первый автомобиль;
   - первый клиент;
   - первая бронь;
   - подсказка подключения GPS;
   - проверка email.

2. Документы и файлы
   - drag-and-drop загрузка;
   - прогресс загрузки;
   - предпросмотр PDF/изображений;
   - теги документов: паспорт, договор, страховка, депозит, возврат;
   - массовая загрузка папки клиента или авто;
   - журнал: кто и когда загрузил файл.

3. Договоры аренды
   - шаблон договора по стране;
   - автоматическая генерация PDF;
   - ссылка клиенту на подпись;
   - статус: draft, sent, viewed, signed;
   - история отправок WhatsApp/email.

4. Финансы
   - раздельный учет аренды, депозитов и возвратов;
   - касса за день;
   - расходы по автомобилю;
   - прибыль по автомобилю;
   - экспорт CSV/XLSX;
   - интеграция Stripe или банковских платежей.

5. GPS и карта
   - реальный Google Maps API key;
   - фильтр по статусу авто;
   - история маршрута;
   - alert при отсутствии сигнала;
   - alert при превышении скорости.

6. Мобильная версия
   - нижняя навигация как в native app;
   - крупные кнопки действий;
   - быстрые сценарии: вернуть авто, принять оплату, загрузить документ;
   - PWA-режим с иконкой на домашнем экране.

7. Командная работа
   - роли owner, manager, accountant;
   - приглашение сотрудников по email;
   - ограничения доступа по разделам;
   - audit log действий.

8. SaaS-монетизация
   - тарифы Starter, Growth, Enterprise;
   - лимиты по автомобилям, пользователям и GPS-устройствам;
   - billing portal;
   - trial 14 дней;
   - usage-based billing для GPS.

## Что стоит убрать или не развивать сейчас

- Дублирующие действия в Settings, если такие же действия уже есть в Bookings, Vehicles, Finance или Service.
- Большие таблицы на мобильном. Лучше карточки и быстрые действия.
- Декоративные кнопки без реального backend-действия.
- Слишком много пунктов меню. Основное меню должно оставаться: Dashboard, GPS, Vehicles, Clients, Bookings, Finance, Service, Settings.

## Следующий технический этап

- E2E-тесты для входа, регистрации, загрузки документа, создания брони, оплаты и возврата депозита.
- Refresh tokens и logout на backend.
- Email verification.
- Password reset.
- Object storage для production-файлов: S3-compatible bucket.
- Rate limiting для auth endpoints.
